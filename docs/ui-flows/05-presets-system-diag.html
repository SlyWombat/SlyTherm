Alarm detail — tap an alarm (plain language + installer code)

‹ Alarm
⚠ Outdoor temperature unavailable

The thermostat can't read an outdoor temperature, so it's playing it safe: the heat pump is paused and heating runs on gas only.

To fix: provide an outdoor temperature — add an outdoor sensor in Home Assistant, or wire an outdoor probe. It clears automatically once a reading arrives.

code: OAT_INVALID · policy: fail-cold (gas only) · source: none

  • Presets — active one is badged + accent-bordered; tap to apply, Edit to change its heat/cool temps; a manual nudge reads as "Hold" until a preset is chosen.
  • System — plain status rows (no "(valid)"): what's running, heat source, the reading + which rooms, outdoor + source, compressor timers, dual-fuel state, gas modulation, furnace-bus state, firmware + uptime.
  • Diagnostics → Alarms: plain-language list (ties to #66) with a drill-in per alarm — what happened, how to fix, and the technical code for an installer.
  • Diagnostics → Bus monitor: the read-only CT-485 sniffer folded in — live decoded frames (time · src→dst · type · bytes) + counters, so the wall unit is also your listen-only bus tool.
  • Diagnostics → Links: WiFi/MQTT/broker + versions (moves the cramped Home status bar's detail here).
