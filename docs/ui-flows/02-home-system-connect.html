Manual entry stays as a fallback. Same design system + entry-is-Status pattern as WiFi. Device screens are 800×480.

  • Discovery-first — Find broker browses mDNS (_mqtt._tcp; fallback _home-assistant._tcp host + default :1883). No IP needed on first run.
  • Found brokers list with name + host + ip:port; empty → "None found" + Rescan and Enter manually… (never a dead end).
  • Confirm & connect prefills host/port from discovery (editable for manual entry); auth is optional (blank = anonymous MQTT).
  • Save/Connect persists to the device (NVS, survives reflash) and connects immediately — no recompile. Entry becomes Status: Connected thereafter.
  • Failure is explicit (auth vs unreachable) with Try again — never a silent spinner.
  • Naming: titled Home Assistant (what you connect to); fields are the MQTT broker. Say if you prefer "Server" or "MQTT".
